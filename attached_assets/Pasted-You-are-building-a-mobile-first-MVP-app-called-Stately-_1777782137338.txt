You are building a mobile-first MVP app called “Stately”.

Stately is an AI condition coach app.
Slogan: “Clarity for your body.”

The app helps users understand their daily physical condition based on activity, sleep, and HRV.
It calculates a daily condition score and gives a simple AI coaching message every morning.

Important:
This is the first MVP version.
Do NOT implement real Apple HealthKit, Samsung Health, or Google Fit integration yet.
Instead, use manual input and mock/sample data.
The structure should be ready for future health data integration.

Use this tech stack:
- Frontend: React Native with Expo
- Language: TypeScript if possible
- Backend: Simple Node.js/Express API or serverless-style API inside the Replit project
- Database: Use local mock data first, but structure the code so it can later be connected to Firebase Firestore
- AI: Prepare an API function for AI coach feedback, but use mock AI responses for now unless an OpenAI API key is provided

Design style:
- Apple Health × Calm inspired
- Minimal, premium, calm, clean
- White background with soft navy and mint sage accents
- Main colors:
  - Navy: #1B2430
  - Mint Sage: #88D3C3
  - Soft Background: #F7F9FA
  - Text Dark: #111827
  - Text Gray: #6B7280
- Use rounded cards, subtle shadows, spacious layout
- Avoid heavy gradients
- The app should feel quiet, trustworthy, and health-focused
- Use SF Pro-like typography if possible; otherwise use system font
- Mobile-first layout

Core user flow:
1. User opens the app
2. Onboarding explains the value of Stately
3. User goes to the home screen
4. User inputs daily data:
   - activity score or active calories
   - sleep score or sleep hours/efficiency/deep sleep
   - HRV value
5. App calculates condition score
6. App shows:
   - today’s condition score
   - status label
   - AI coach advice
   - mini 7-day graph
7. User can view history
8. User can open settings

3. Daily Input Screen
Allow user to input:
- Active calories
- Goal calories
- Sleep hours
- Sleep efficiency percentage
- Deep sleep percentage or deep sleep score
- HRV value
- HRV reliability: Green / Amber / Red

When user taps “Calculate Condition”:
Calculate:
activity_score = clamp((activeKcal / goalKcal) * 100, 0, 120)

sleep_score = 
0.20 * sleep_time_score + 
0.45 * sleep_efficiency_score + 
0.35 * deep_sleep_score

For MVP:
- Convert sleep hours to sleep_time_score:
  - 8 hours or more = 100
  - 6 hours = 75
  - 5 hours = 60
  - under 5 hours = 45
- Use sleep efficiency directly as sleep_efficiency_score
- Use deep sleep input directly as deep_sleep_score

For HRV score in MVP:
Use a simplified version:
- HRV >= 70: 90
- HRV >= 55: 75
- HRV >= 40: 60
- HRV >= 30: 45
- HRV < 30: 30

Final condition score:
condition = 0.30 * activity_score + 0.40 * sleep_score + 0.30 * hrv_score

If HRV reliability is Red:
Adjust weights:
condition = 0.35 * activity_score + 0.50 * sleep_score + 0.15 * hrv_score

If HRV reliability is Amber:
Adjust weights:
condition = 0.32 * activity_score + 0.43 * sleep_score + 0.25 * hrv_score

Clamp final score between 0 and 100.

Save the calculated result to local state/mock history.

AI coach logic:
For MVP, generate advice with rule-based mock logic.

Rules:
- If condition score >= 85:
  “Your body looks well recovered today. A productive training day is possible.”
- If condition score >= 70:
  “You are in good condition. Keep your workout moderate and stay hydrated.”
- If condition score >= 50:
  “Your body shows some fatigue. Choose light cardio or mobility work today.”
- If condition score < 50:
  “Recovery should be your priority today. Reduce training intensity and focus on sleep.”

Also consider weakest metric:
- If sleep score is lowest:
  mention sleep recovery
- If HRV score is lowest:
  mention stress, caffeine, or relaxation
- If activity score is too high:
  mention accumulated fatigue
- If activity score is too low:
  mention light movement
